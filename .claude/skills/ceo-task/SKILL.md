# 대표 지시사항 자동 처리

사용자가 대표 카톡 지시 원문을 붙여넣으면 아래 순서로 처리한다.

## 1단계: 지시 분류

| 지시 패턴 | 워크플로 |
|---|---|
| "N탄 올려주세요" / "이미지·설명문 올려놨음" | `series-upload` 스킬 실행 |
| "홈페이지 ~수정" / "~삭제" / "~변경" | 홈페이지 수정 (아래 규칙) |
| "색인" / "검색 안 나옴" / "노출" | SEO 점검 (아래 규칙) |
| "SNS에 올려주세요" | SNS 문안 생성 (직접 게시는 불가 — 문안+체크리스트 제공) |
| "고객DB 기록" / 문의 전화·미팅 결과 | 고객DB 기입 항목 정리 (아래 템플릿) |
| "품의서/검수확인서 작성" | 이지바로 서류 초안 (기존 파일 양식 재사용) |
| 영문 홈페이지 관련 | 국문과 동일 워크플로 + 영문 콘텐츠 검증 |

분류가 애매하면 실행 전에 1문장으로 해석을 확인받는다. 단, 명확하면 바로 실행.

## 2단계: 실행 규칙 (대표 스타일 반영 — 대화록에서 학습된 것)

- **속도 우선**: 대표는 당일 완료를 기대. 완벽한 100%보다 "우선 반영 + 후속 다듬기" 순서.
- **우선순위 충돌 시**: 대표 지시 > 실장(김낙순) 요청 > 정기 업무. 이지바로 입력일(화·목 오전 9시)과 겹치면 상황 보고 후 대표 지시 먼저.
- **보고는 짧게**: 지시 접수엔 "네, 알겠습니다"만. 장황한 계획 설명 금지 (대표가 5/27 "말이 참 많구만" 지적).
- **완료 보고는 결과만**: 뭘 했는지 나열이 아니라 "반영 완료 + 확인 방법" 중심.
- **보안**: 내부 자료(영업가이드·상세 사업성 자료·기술 세부)는 홈페이지/SNS에 절대 노출 금지. 이천 외관 등 건물 사진, 개인 이름 노출 금지. 대표 연락처는 왓츠앱 번호만.
- **영문 사이트에 한글 콘텐츠 노출 금지** — 영문 이미지 없으면 해당 콘텐츠는 영문에서 제외.

## 3단계: 산출물 (항상 이 3종 세트)

1. 변경 내역 요약 (무엇을 어디에)
2. **복사-붙여넣기용 커밋 메시지** (Summary 1줄 + Description 불릿)
3. **카톡 보고 문구** (아래 형식)

```
[홈페이지 수정 보고 — M/D]

말씀하신 내용 반영 완료했습니다.
① (지시 요약) → (처리 결과)
② ...

배포 완료, 라이브 반영 확인했습니다.
```

## 고객DB 기입 템플릿 (문의 전화·미팅 시)

```
날짜 | 회사/이름 | 연락처 | 유입경로(블로그/홈페이지/소개) | 관심모델(알파팜코어/카페/컨테이너/쿨링) |
보유자산(건물/창고/담보 여부 ← 대표가 가장 중시) | 요청사항 | 다음 액션 | 가능성 평가
```
대표 기준: "창고나 건물이 있고 담보능력이 있는 사람이 중요. 땅만 있다는 사람은 거의 가능성 없음."

## 라이브 검증 (배포 후)

`curl.exe -s https://www.ceslprimus.com/[경로]` 로 title/이미지/텍스트 반영 확인 후 보고에 "라이브 확인 완료" 명시.
